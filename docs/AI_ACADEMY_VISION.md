# AI Academy — Vision produit et pédagogique

## 1. Vision

AI Academy n’est pas une simple plateforme de cours. Elle doit devenir une véritable école de développement assisté par l’intelligence artificielle.

Son objectif est d’aider un vrai débutant à devenir capable de créer, comprendre, modifier, tester et publier une application simple avec l’aide d’outils comme Claude Code, Codex, Make et des agents IA.

L’utilisateur ne doit pas seulement lire des explications. Il doit apprendre en construisant, en prenant des décisions, en testant, en corrigeant et en validant des compétences concrètes.

## 2. Mission

Rendre accessible le développement assisté par IA à une personne non technique, sans la noyer dans le jargon ni lui faire copier du code sans compréhension.

AI Academy doit permettre à l’apprenant de :

- comprendre les grandes briques d’une application ;
- savoir ce qu’il doit réellement maîtriser ;
- savoir ce qu’il peut déléguer à l’IA ;
- savoir vérifier et corriger le travail produit par l’IA ;
- avancer par petites versions testables ;
- construire progressivement de vraies applications.

## 3. Public cible

AI Academy s’adresse en priorité à :

- des adultes débutants en développement ;
- des personnes qui souhaitent créer une application sans devenir développeur expert ;
- des porteurs de projet qui veulent utiliser l’IA comme copilote ;
- des professionnels souhaitant apprendre l’automatisation et les agents IA ;
- des personnes qui ont besoin d’une pédagogie très concrète, progressive et rassurante.

## 4. Promesse pédagogique

À la fin d’un parcours, l’apprenant ne doit pas seulement « connaître » une notion.

Il doit être capable de :

- l’expliquer avec ses propres mots ;
- reconnaître où elle apparaît dans une application ;
- savoir quoi modifier lui-même ;
- savoir quoi demander à Claude Code ou Codex ;
- vérifier que la modification fonctionne ;
- produire un livrable concret.

## 5. Principes pédagogiques obligatoires

### 5.1 Une compétence réelle par module

Chaque module doit développer une compétence concrète et vérifiable.

Un module ne se valide pas uniquement avec un quiz. Il se valide par une combinaison de compréhension, manipulation et production.

### 5.2 Une notion principale par leçon

Chaque leçon doit traiter une seule idée principale.

Les explications doivent être courtes, simples et techniquement exactes.

### 5.3 Aucun jargon sans définition

Chaque terme technique doit être défini immédiatement avec des mots simples et un exemple concret.

### 5.4 Comprendre avant de copier

L’apprenant ne doit jamais copier du code sans savoir à quoi il sert.

Chaque démonstration doit distinguer :

- ce qu’il faut comprendre ;
- ce qu’il faut savoir modifier ;
- ce que l’IA peut gérer ;
- ce qu’il faut vérifier après l’intervention de l’IA.

### 5.5 Pratiquer davantage que lire

L’expérience doit alterner :

- explication courte ;
- schéma ;
- démonstration ;
- manipulation ;
- exercice ;
- quiz ;
- mini-défi ;
- mini-projet ;
- évaluation pratique ;
- révision visuelle.

### 5.6 Apprendre en construisant

Chaque niveau doit conduire à un livrable réel.

Le fil rouge du premier parcours est une application de suivi d’apprentissage, puis le projet final est Wayli.

### 5.7 Avancer par petites versions testables

Aucun développement massif ne doit être réalisé sans étape intermédiaire.

Chaque fonctionnalité doit être découpée, testée et validée avant d’élargir le périmètre.

## 6. Modèle d’un module AI Academy

Chaque module doit tendre vers la structure suivante :

1. Introduction et utilité concrète
2. Compétence cible
3. Prérequis
4. Leçons courtes
5. Schémas interactifs
6. Démonstrations guidées
7. Manipulations réelles
8. Exercices avec indices
9. Quiz avec explications détaillées
10. Mini-défi
11. Mini-projet
12. Révision visuelle
13. Évaluation pratique
14. Validation de compétence

## 7. Expérience recherchée

AI Academy doit être :

- intuitive ;
- moderne ;
- agréable sur mobile et ordinateur ;
- peu chargée visuellement ;
- adaptée à un vrai débutant ;
- rapide à installer et à maintenir ;
- centrée sur la pratique ;
- rassurante sans être infantilisante.

L’expérience peut s’inspirer de Brilliant, Codecademy et Duolingo, sans copier leur identité visuelle ni leurs contenus.

## 8. Progression par compétences

La progression ne doit pas seulement afficher des modules terminés.

Elle doit progressivement permettre de visualiser :

- les compétences acquises ;
- les compétences en cours ;
- les points à renforcer ;
- les projets réalisés ;
- les évaluations réussies ;
- les livrables produits.

Exemple :

- HTML : acquis
- CSS : acquis
- JavaScript : en cours
- TypeScript : à renforcer
- React : non commencé

## 9. Rôle de l’IA dans l’apprentissage

L’IA doit être utilisée comme copilote, pas comme remplaçante de l’apprenant.

Elle peut :

- expliquer ;
- proposer un plan ;
- générer du code ;
- corriger une erreur ;
- produire un test ;
- automatiser une tâche ;
- agir comme développeur ou agent.

Mais l’apprenant doit toujours :

- définir l’objectif ;
- vérifier les changements ;
- tester ;
- comprendre le résultat ;
- décider d’accepter, corriger ou refuser.

## 10. Parcours prévus

### Parcours 1 — Créer une application avec Claude Code

Objectif : apprendre à concevoir, comprendre, modifier, tester et publier une application simple.

### Parcours 2 — Automatiser avec Make

Objectif : connecter des outils et automatiser des tâches répétitives sans code complexe.

### Parcours 3 — Créer des agents IA

Objectif : comprendre ce qu’est un agent, à quoi il sert, comment le concevoir, lui donner des outils, le sécuriser et le superviser.

Ce parcours doit notamment couvrir :

- différence entre chatbot, assistant et agent ;
- mémoire ;
- outils ;
- autonomie ;
- limites ;
- conception d’un agent ;
- création d’un premier agent ;
- multi-agents ;
- orchestration ;
- déclenchement planifié ;
- validation humaine ;
- projet final multi-agents.

### Parcours 4 — Développer avec des agents IA

Objectif : utiliser plusieurs agents pour piloter un cycle complet de développement.

Exemples :

- agent product manager ;
- agent développeur ;
- agent testeur ;
- agent relecteur ;
- agent déployeur ;
- GitHub Actions ;
- tâches planifiées ;
- pull requests automatiques ;
- supervision humaine.

## 11. MVP actuel

Le MVP doit rester simple.

Il contient :

- une page d’accueil ;
- un tableau de bord ;
- un parcours « Créer une application avec Claude Code » ;
- la liste des modules ;
- des leçons courtes ;
- des schémas ;
- des quiz ;
- des exercices avec indices et correction ;
- une progression locale ;
- une navigation précédent/suivant ;
- une interface responsive.

## 12. Fonctionnalités exclues du MVP

Ne pas ajouter prématurément :

- paiement ;
- espace administrateur ;
- réseau social ;
- badges complexes ;
- certificat ;
- chatbot intégré ;
- authentification ;
- base de données distante ;
- plusieurs parcours complets en parallèle ;
- fonctionnalités décoratives non indispensables.

## 13. Stratégie de construction

### Phase 1 — Terminer le socle

Finaliser les 12 modules du premier parcours avec leur contenu complet.

### Phase 2 — Construire la V2 pédagogique

Ajouter progressivement :

- compétences ;
- défis ;
- simulations ;
- mini-projets ;
- évaluations pratiques ;
- révisions visuelles ;
- suivi détaillé des acquis.

### Phase 3 — Enrichir les modules existants

Reprendre les Modules 1 à 12 un par un sans jeter le contenu existant.

Ajouter la couche pédagogique premium autour du socle déjà produit.

### Phase 4 — Ajouter les nouveaux parcours

Déployer progressivement les parcours Make, Agents IA et Développement avec agents IA.

## 14. Règles de travail pour les assistants IA

Avant toute modification importante, Claude Code, Codex ou tout autre agent doit :

1. lire ce document ;
2. inspecter le projet existant ;
3. proposer la solution la plus simple ;
4. distinguer ce qui est certain, supposé et à valider ;
5. travailler sur une branche séparée ;
6. avancer par petites versions testables ;
7. exécuter les tests disponibles ;
8. ne jamais fusionner automatiquement dans main sans validation humaine ;
9. ne pas ajouter de fonctionnalité hors périmètre ;
10. produire un compte rendu clair.

## 15. Critères de réussite

AI Academy réussit si un vrai débutant peut :

- suivre un parcours sans se sentir perdu ;
- comprendre les notions essentielles ;
- pratiquer plus qu’il ne lit ;
- créer une application simple ;
- utiliser Claude Code et Codex avec méthode ;
- repérer une mauvaise proposition de l’IA ;
- corriger un problème simple ;
- publier un projet ;
- concevoir ensuite des automatisations et des agents IA.

## 16. Principe directeur

AI Academy ne doit pas seulement transmettre des connaissances.

Elle doit faire acquérir des compétences.

L’objectif n’est pas que l’apprenant sache réciter ce qu’est une application, un composant, une API ou un agent IA.

L’objectif est qu’il sache les utiliser pour construire quelque chose de réel, avec l’aide de l’IA, tout en gardant le contrôle du projet.
