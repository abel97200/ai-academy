# Curriculum — « Créer une application avec Claude Code »

Formation autonome organisée en **4 niveaux** et **12 modules**. Le fil rouge pratique est un **tracker d'apprentissage** (une to-do list de leçons) ; le projet final du cursus est **Wayli**, une application qui guide un débutant dans un parcours de vie ou une démarche complexe (étapes, conseils, checklists, progression).

Statut actuel : **seul le Module 1 a un contenu complet**. Les modules 2 à 12 n'ont que leurs métadonnées (titre, objectif, niveau, prérequis, compétences) et sont marqués « à venir ».

---

## Niveau 1 — Comprendre et préparer

| # | Module | Objectif |
|---|---|---|
| 1 | Comprendre comment fonctionne une application | Expliquer le trajet d'une action utilisateur jusqu'à l'affichage et à l'enregistrement d'un résultat |
| 2 | Transformer une idée en projet réalisable | Passer d'une idée floue à un projet cadré et réalisable |
| 3 | Préparer son environnement de travail | Installer et comprendre les outils nécessaires pour développer |

**Projet de niveau : cadrer et démarrer le tracker.**

---

## Niveau 2 — Comprendre et modifier le code utile

| # | Module | Objectif |
|---|---|---|
| 4 | Construire une interface avec HTML | Structurer une page web avec HTML |

| 6 | Ajouter de la logique avec JavaScript et TypeScript | Rendre une interface interactive et fiable |

| # | Module | Objectif |
|---|---|---|

**Projet de niveau : transformer le tracker en application complète.**

## Niveau 4 — Tester, publier et devenir autonome

| # | Module | Objectif |
|---|---|---|
| 10 | Piloter Claude Code efficacement | Collaborer efficacement avec un copilote IA |
| 11 | Tester et corriger une application | Repérer, comprendre et corriger des problèmes |
| 12 | Publier, maintenir et améliorer | Mettre une application en ligne et la faire évoluer |

**Projet de niveau : finaliser et publier le tracker.**

---

## Projet final du cursus — Wayli

1. Cadrer une version simple de Wayli.
2. Définir son MVP (produit minimum viable).
3. Construire un parcours guidé.
4. Enregistrer la progression.
5. Tester.
6. Corriger.
7. Publier l'application.

## Règles de validation

- **Une leçon** est terminée quand son bloc `validation` a été validé (comme aujourd'hui).
- **Un module** est validé quand :
  - toutes ses leçons sont terminées ;
  - le meilleur score obtenu à chacun de ses quiz atteint au moins **75 %** ;
  - chaque action pratique obligatoire du module a été confirmée ;
  - chaque évaluation finale (`assessment`) du module a été confirmée.
- **Un niveau** est validé quand tous ses modules sont validés (le projet de niveau est affiché à titre indicatif ; sa validation interactive n'est pas encore implémentée — voir limites en fin de livraison).
- Un module dont la liste de leçons est vide est **« à venir »**. Un module qui a du contenu mais dont les prérequis (autres modules) ne sont pas encore validés est **« verrouillé »**.
